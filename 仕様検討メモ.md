# 熱気球フライトゲーム 仕様検討メモ

国土地理院の地形データを読み込んで遊ぶゲームの仕様検討メモ。
仕様は確定済み。**プロトタイプ実装に着手済み**(`prototype/` フォルダ)。

## プロトタイプ現況(2026-07-08)

- 場所: `prototype/index.html` + `main.js` + `terrain.js`(ブラウザ+Three.js、ビルド不要)
- 起動: `python -m http.server 8000 --directory c:\Claude\game` → http://localhost:8000/prototype/
- 実装済み: 佐賀・嘉瀬川周辺20km四方の地形(DEM z13+写真z14)、バーナー(Space長押し)/リップライン(R長押し)、
  高度別風レイヤー(パイバルプリセット固定)、計器HUD、パイバル表、時間加速×1〜×8、視点切替(V)、燃料、出典表記
- 実装済み(JDG): マーカー投下(M、1本)。重力+空気抵抗(終端速度10m/s)で落下し落下中も風に流される。
  ターゲット(オレンジX+白リング、エリア中央)、着地点→ターゲットの距離をリザルト表示、
  自己ベストをlocalStorageに保存、HUDにターゲット距離/方位を常時表示
- 実装済み(競技の流れ): タスクブリーフィング画面(タスクシート+パイバル表+標準地図)。
  地図はズーム(ホイール、z11〜z17タイル切替)・パン(ドラッグ)対応のスリッピーマップ。
  地図クリックで離陸地点を自由選択(個別離陸)→離陸ボタンで飛行開始。
  制限時間30:00(ゲーム内時間)をHUDでカウントダウンし、時間内に投下できなければ現在地点計測にフォールバック
- 実装済み(風): ブリーフィングでパイバルを手入力可能(行の追加削除・プリセット4種・数値編集)。
  風テーブルは `?w=ft,dir,kt;…` 形式でURLに載り、「条件URLをコピー」ボタン(ブリーフィング/リザルト)で共有可能。
  風向補間は最短角度経路(350°→020°は北回り)に対応
- 実装済み(地形LOD): 気球の近く(3km以内)の地形タイルを裏でz16写真(約2m/px、従来の4倍精細)に
  段階アップグレード(1.5秒に1枚、上限12枚)。離陸地点とターゲットのタイルは離陸時に即時高解像度化。
  デバッグ用 `?autostart=1` でブリーフィングを飛ばして即離陸可能(自動テスト用)
- 実装済み(地形LOD第2段): 対地高度1000m未満のとき、気球直下の1タイルのみz17(約1m/px、4096px)へ
  さらにアップグレード。直下タイルが変わると前のタイルはz16に戻し、z17はメモリ上常に1枚だけ保持(実機確認済み)
- 実装済み(エリア選択): 起動時に日本全図のスリッピーマップでエリア中心をクリック選択

  ターゲットは選択エリアの中央。海上など標高タイル欠損は標高0として処理(クラッシュしない)
  ※`?a=`直行→ブリーフィングの実機確認は未了(ヘッドレス不安定のため)。選択画面表示は確認済み

## 決定事項

- **ゲーム内容**: 実在の日本の地形を3D化した空間で熱気球を操縦するフライトゲーム
- **対象エリア**: ゲーム開始前にプレイヤーが地域を選択できる(日本全国から自由に選択)
  - → 地理院タイルを動的に取得する仕組みが必要
- **ゲームの目的**: 熱気球競技の競技規定に基づく **JDG(Judge Declared Goal)競技** を行う
  - 競技委員(ゲーム側)が設定したゴール(ターゲット)にマーカーを投下し、ターゲット中心からの距離を競う
- **風のモデル**: 高度ごとに風向・風速を設定するゲーム的レイヤーモデル
  - 低〜中高度: **パイバル(パイロットバルーン)観測データを入力**して風向・風速を定義する
  - 高高度: 天気図(気圧配置)の影響を受けた風として設定
- **リアリティの方向性**: 中間(操作は簡単、浮力の慣性や着陸・投下判定はそれなりにリアル)
- **競技タスクの範囲**: まずJDG単体で完成させる。ただしタスク定義はデータ化しておき、将来PDG・FON・HWZ等を追加しやすい設計にする
- **離陸地点**: タスク設定で「個別離陸(プレイヤーがエリア内で自由に選択)」「共通ローンチサイト固定」の両方を指定できるようにする(実競技に近い形)
- **マーカー投下**: 物理落下を簡易再現。重力+空気抵抗で落下し、落下中も風に流される(低高度から投げるほど正確、という実競技の駆け引きを再現)
- **パイバルデータ入力**: 高度ごとの風向・風速をテーブルで手入力できる+典型的な風パターンのプリセットも用意
- **入力単位**: 実競技慣行に合わせる(高度はフィート刻み〔例: 500ft〕、風向は磁方位度数、風速はノット)。表示単位は設定でm/s系に切替可
- **スコアリング**: マーカー着地点からターゲット中心までの距離(m)をそのままリザルトとする。マーカー未投下時は着陸地点で計測
- **飛行中の計器・情報表示**: 以下をすべて表示する
  - 基本計器(高度計・昇降計〔バリオメーター〕)
  - 地理院地図+自機位置・軌跡・ターゲット表示(GPS相当)
  - 現在高度の風向・風速
  - ゴール(ターゲット)の位置
  - 入力済みパイバル表の参照
- **フライト前フェーズ**: 実競技のブリーフィングを再現する
  - タスクシート提示(ターゲット座標・離陸条件・制限時間)→ パイバルデータ確認/入力 → 地図で離陸地点選択 → 離陸
- **時間の扱い**: 基本は実時間スケール+プレイヤーが任意に×2〜×8の時間加速を使える(風読みのリアリティを保ちつつ1プレイを短縮)
- **視点**: ゴンドラからの一人称視点が基本、俯瞰確認用に三人称・真下視点へ切替可
- **競技相手**: まずはソロのスコアアタック。同一タスク・同一風での自己ベストや過去記録との比較機能を持たせる
- **制限時間・終了条件**: タスクシートで制限時間(スコアリングピリオド)を指定。時間内にマーカー投下できなければ着陸地点計測にフォールバック
- **操作方式**: いずれも長押し操作
  - **バーナー**: 長押しで噴射(加熱の効きは数秒遅れて現れる)
  - **排気**: リップラインを長押しで引き、気球頂部のパラシュート(リップパネル)が開いて排気する方式(ベント方式ではない)
- **燃料**: プロパン残量表示あり。消費はバーナー噴射時間に比例するシンプルなモデル(残量が実質の飛行可能時間になる)
- **記録保存・共有**: 自己ベストはブラウザのlocalStorageに保存。風+タスクの条件はURLパラメータ(シード)にエンコードし、URL共有だけで同条件に挑戦できる(サーバー不要)
- **対象エリアの広さ**: 1タスクあたり15〜20km四方(実競技の飛行エリアに近いスケール。DEM10B+必要に応じ高精度化)
- **プラットフォーム**: ブラウザ+Three.js に決定
  - 地理院タイルの動的取得と最も相性が良い(CORS対応済み・fetchでそのまま取得可)
  - 「URLを開くだけで同条件(風+タスク)に挑める」配布・共有性がソロ+記録比較の設計と噛み合う
  - 将来Electronでデスクトップ版に包む移行も容易

## 使用予定の国土地理院データ(整理済み)

- **標高タイル(DEM)**: PNG標高タイル。DEM5A(5mメッシュ・高精度)/ DEM10B(10mメッシュ・全国カバー)。地形メッシュ生成の核
- **地理院タイル(画像)**: 標準地図・淡色地図・航空写真・陰影起伏図など。地形テクスチャに使用
- **ベクトルタイル**: 道路・河川・建物輪郭など(必要に応じて)
- 利用規約: 出典明記(「国土地理院タイル」等)で商用含め利用可

## 熱気球のゲーム性(前提の共有)

気球は「バーナーで上昇・冷却/排気で下降、水平移動は風任せ」という乗り物。
→ **高度によって風向・風速が変わる中を、上下操作だけで目的地を目指す**というゲーム性が自然に成立する。
